📈 Opciones BYMA — Guía de uso

Programa paso a paso para aprender opciones del mercado argentino: wiki de estudio + simulador interactivo + backtesting, con un companion de IA al lado.

1 · Qué es este proyecto

Un monorepo con dos piezas que se retroalimentan:

📚 La wiki (content/wiki/ → /aprender)

El programa de estudio en español, en módulos ordenados. Cada lección tiene teoría, ejemplos del mercado argentino, ejercicios y fuentes. Los archivos son markdown plano: también podés abrir content/ como bóveda de Obsidian.

🧮 El simulador (/simulador)

Armás posiciones con calls, puts y acciones, y ves el diagrama de payoff, breakevens, ganancia/pérdida máxima y griegas. Primero con datos manuales e históricos; más adelante, conexión en vivo con un broker.

2 · Cómo arrancar

# requisitos: Node ≥ 20 y pnpm (npm install -g pnpm)
cd ~/Projects/opciones-byma
pnpm install
pnpm dev          # abre http://localhost:3000
pnpm test         # corre los tests del motor de cálculo

La app levanta con la wiki ya navegable. El botón "Empezar por la primera lección" te lleva al módulo 00.

4 · Mini chuleta: leer un ticker

GFGC4700JU

GFGSubyacente: GGAL (el prefijo no es el ticker de la acción)
CC = call · V = put ("venta")
4700La base (precio de ejercicio)
JUVencimiento: junio, 3er viernes. Ciclo bimestral: FE·AB·JU·AG·OC·DI

Lote = 100 acciones. La prima cotiza por acción: 1 lote a prima $205 = $20.500.

5 · El companion de IA

El repo incluye un CLAUDE.md con todo el contexto del dominio (glosario argentino, estructura, convenciones). Abrí una sesión de Claude Code dentro del proyecto y usalo como tutor:

  • "Explicame el ejercicio 3 de la lección de la prima, no me sale."
  • "Calculá con packages/core el delta de un call GGAL base 5300 a 14 días con IV 55%."
  • "¿Por qué mi lanzamiento cubierto tiene este breakeven? Verificalo."

Más adelante (fase M4) el simulador suma un panel de chat integrado que ve tu posición actual.
